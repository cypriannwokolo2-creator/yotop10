# M3.5 — Anonymous Users

> **Goal**: A visitor can interact with YoTop10 without creating an account. They get a persistent anonymous identity tied to their device.

---

## Overview

Anonymous users can browse, react, comment, and post (to review queue). They can also create communities but need a real account as moderator. This dramatically increases top-of-funnel activity.

---

## Device Fingerprinting

For maximum security and uniqueness, use ALL of these signals:
- Canvas fingerprint (GPU rendering)
- WebGL fingerprint (graphics capabilities)
- Audio context fingerprint
- Screen resolution + color depth
- Timezone + language
- Installed fonts (detected via JS)

Combine into a hash for the device fingerprint.

---

## Session Management

- **Session duration**: 12 hours after NO activity
- Activity = any click, scroll, or interaction
- If user is just reading without interacting, it counts as NO activity

### New Device Login
- When anonymous user logs in from new device:
- Old session is IMMEDIATELY deleted
- New device is tracked strictly
- No multiple simultaneous sessions

---

## Visual Indicator

Anonymous users should be clearly identifiable:
- **Icon**: Ghost/mask icon next to username
- **Username color**: Muted gray (not bright)
- **Tooltip on hover**: "Anonymous Account"

This is less stigmatizing while still being transparent.

---

## Post Review Flow

1. Anonymous posts go to **review queue** (like all posts)
2. If **approved**: Goes to public feed
3. If **rejected**: Automatically goes to author's private feed
4. Connections can turn off your posts in their private feed

---

## Verified Authors

- Verified authors (90+ trust score): Posts go live first → reviewed AFTER
- This is a privilege for trusted users
- See trust-score.md for full details

---

## Rate Limiting

- New accounts (< 3 months) have slightly higher rate limits
- Rate limiting removed after 3 months of consistent activity
- Active = at least 1 action per week

---

## Account Conversion

### What DOESN'T Happen:
- NO content migration from anonymous to registered account
- Accounts remain SEPARATE entirely
- Content on anonymous account stays anonymous

### What Happens:
- When anon user signs up → they get a NEW registered account
- Their anonymous content remains on the anonymous account
- They start fresh with the new account

---

## Spam Prevention

- Device fingerprinting for identity
- VPN detection → block immediately
- Same IP (office/school) is OK → fingerprint differentiates users
